# Jury Q&A

**Почему table-first?** Полный PDF parsing слишком рискован для дедлайна; таблица фактов дает проверяемый граф и быстрый UI.

**Как снижаете hallucination?** Accepted claim требует `document_id`, locator, quote, confidence. Без цитаты факт не считается подтвержденным.

**Где здесь LLM?** Базовый поиск выполняется live и детерминированно: вопрос парсится, сущности линкуются, SQLite-граф запрашивается, ответ собирается из проверенных фактов. При заполненных `YANDEX_API_KEY` и `YANDEX_FOLDER_ID` в сайдбаре включается `LLM-резюме (YandexGPT)`: `yandexgpt-5.1` формулирует краткий текст только поверх найденных фактов. Если API недоступен, демо не падает и показывает обычный проверяемый ответ.

**Почему YandexGPT 5.1?** Это default для sanity-probe на русском техническом extraction-сценарии; Qwen3 указан fallback. Полный benchmark не блокирует демо.

**Это заранее записанные ответы?** Нет. Подготовлены 12 проверенных демо-вопросов в UI, чтобы жюри быстро увидело три сценария: главный ответ, пробелы и недостаточность данных. При нажатии `Найти ответ` заново выполняется deterministic retrieval: парсинг вопроса, entity linking, запрос к SQLite-графу и сборка ответа из фактов.

**Зачем pills под radio?** Radio выбирает сценарий демо, pills выбирают конкретный вопрос внутри сценария. Поле `Вопрос` остается редактируемым, и кнопка всегда отвечает на текущий текст поля.

**Что если Docker не работает?** Есть официальный fallback: `python -m pip install -r requirements.txt` и `streamlit run app.py --server.port 8501`.

**Можно ли масштабировать?** Да: расширить curated manifest, заменить SQLite/NetworkX на Neo4j/Qdrant profile и добавить batch extraction.
