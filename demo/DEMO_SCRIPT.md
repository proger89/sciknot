# Demo Script

1. Откройте `http://localhost:8501`.
   Если `.env` заполнен Yandex-доступами, в сайдбаре можно включить `LLM-резюме (YandexGPT)`. Для стабильной записи сначала покажите базовый режим без LLM, затем включите переключатель и повторите тот же вопрос.
2. Покажите чистое стартовое состояние: есть метрики, примеры вопросов, пустое поле и кнопка `Найти ответ`; ответа ниже еще нет.
3. Выберите radio `Главный ответ`: первый вопрос категории подставляется в поле, ниже появляются pills с вариантами.
4. Кликните pill `Плотность наблюдений`: поле меняется, но ответ не появляется автоматически. Это показывает, что UI не раскрывает заготовку, а только выбирает вопрос.
5. Нажмите `Найти ответ`: покажите стадии `Разбираю вопрос`, `Найдены сущности`, `Ищу связанные факты`, затем строку `Пайплайн ... мс`.
6. В первом ответе проговорите: вопрос обработан live; найден факт с источником, а система честно показывает confidence и пробелы.
7. Откройте/покажите блоки: таблица фактов, источники и цитаты, пробелы, карта связей ответа. JSON-экспандер оставьте как техническое подтверждение структуры.
8. Выберите `Пробелы`, затем pill `Верификация прогноза`, нажмите `Найти ответ`: покажите пробелы, источники и карту связей.
9. Выберите `Недостаточно данных`, нажмите `Найти ответ`: покажите, что система умеет отвечать не только "да", но и честно помечать недостаточность.
10. Введите чужой вопрос, например `Что известно про климатические выбросы?`, нажмите `Найти ответ`: UI не падает, а показывает известные сущности демо-графа.
11. Кликните по сущности `Комсомольский рудный массив`: вопрос подставляется в поле. Это демонстрирует graceful fallback вместо traceback.
12. Завершите фразой: подготовлены 12 проверенных вопросов, но сам pipeline выполняется при клике; предвычислен только индекс.
13. Если включали `LLM-резюме`, отдельно проговорите: YandexGPT не ищет факты и не заменяет граф, а только сжимает уже найденные доказательства; при ошибке API система возвращается к детерминированному ответу.
