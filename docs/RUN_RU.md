# Пошаговый запуск

## Вариант A: Docker

1. Скопируйте `.env.example` в `.env`.
2. Для обычного демо `.env` можно оставить пустым: UI будет работать в детерминированном GraphRAG-режиме без LLM. Если заполнить `YANDEX_API_KEY` и `YANDEX_FOLDER_ID`, в сайдбаре появится переключатель `LLM-резюме (YandexGPT)`.
3. Запустите:

```powershell
.\scripts\start_demo.ps1
```

Скрипт соберет Docker-образ, запустит сервис и сам откроет `http://localhost:8501` в браузере. Если Docker недоступен, он установит зависимости и запустит локальный Streamlit fallback.

При запуске приложение готовит демо-данные автоматически: обновляет manifest корпуса `data/raw_manifest.csv`, собирает SQLite-граф `data/processed/sciknot.sqlite` и snapshot `data/processed/graph_snapshot.json`, если артефактов нет.

Ручной Docker-запуск без автооткрытия браузера:

```powershell
docker compose up --build
```

## Опционально: LLM-резюме YandexGPT

1. Откройте `.env`.
2. Заполните только локально:

```dotenv
YANDEX_API_KEY=ваш_ключ
YANDEX_FOLDER_ID=ваш_folder_id
```

3. Перезапустите Streamlit или Docker.
4. В сайдбаре включите `LLM-резюме (YandexGPT)`.
5. Нажмите `Найти ответ`: поиск фактов всё равно идёт детерминированно по SQLite-графу, а YandexGPT только формулирует краткое резюме строго по найденным фактам. Если API недоступен, UI автоматически оставит обычный детерминированный ответ.

Sanity-probe текущей `.env` проверил `yandexgpt-5.1`: default-модель отвечает HTTP 200. `qwen3-235b-a22b-fp8` оставлен в конфиге как fallback, но текущий probe вернул API blocker HTTP 400, поэтому для демо выбран `yandexgpt-5.1`.

## Как пользоваться UI

1. В блоке `Демо-вопрос` выберите один из трех сценариев: `Главный ответ`, `Пробелы` или `Недостаточно данных`.
2. Radio-кнопка подставит первый вопрос категории в поле `Вопрос`. Ниже появятся pills `Варианты вопроса`: это остальные проверенные вопросы этой категории.
3. Выберите pill, если нужен другой вариант. Всего подготовлено и протестировано 12 вопросов: 5 главных, 4 про пробелы, 3 про недостаточность данных.
4. Ответ еще не показывается: это нормальное пустое состояние. Нажмите `Найти ответ` или Enter в поле вопроса.
5. Кнопка запускает live deterministic retrieval pipeline: разбор вопроса, поиск сущностей, запрос к графу, сборку доказательств. Подготовлен только индекс.
6. Если меняете текст вопроса вручную, кнопка всегда отвечает на текущий текст поля `Вопрос`, а не на выбранную radio-кнопку или pill.
7. В ответе смотрите блоки сверху вниз: краткий вывод, скорость и число источников, уверенность извлечения, таблица фактов, источники и цитаты, пробелы, карта связей ответа.
8. Если вопрос не сопоставился с демо-графом, UI покажет известные сущности. Клик по сущности подставит новый вопрос в поле.
9. `Технический ответ (JSON)` нужен только для проверки внутренней структуры ответа.

## Вариант B: без Docker

1. Установите зависимости:

```powershell
python -m pip install -r requirements.txt
python scripts/prepare_demo_data.py
```

2. Запустите UI:

```powershell
streamlit run app.py --server.port 8501
```

Откройте `http://localhost:8501`.

## Остановка

Docker: `docker compose down`.

Локально: остановите процесс Streamlit в терминале через `Ctrl+C`.
